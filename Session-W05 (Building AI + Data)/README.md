# Session W05 — Building AI + Data

Azure SQL AI capabilities demo: vector embeddings, text chunking, semantic search, DiskANN vector indexes, change tracking, Data API Builder (DAB) with MCP, and Azure Functions.

## Architecture

- **Azure SQL** (GP Serverless, centralus) — AiDemoDb with VECTOR(1536), AI_GENERATE_EMBEDDINGS, AI_GENERATE_CHUNKS, VECTOR_SEARCH
- **Azure OpenAI** — text-embedding-3-large (1536 dimensions)
- **Data API Builder** — REST, GraphQL, and MCP endpoints for all tables/views/procs (Azure Container Apps)
- **Azure Function** — SQL trigger binding on ReviewVector change tracking

## Prerequisites

- Azure CLI (`az`)
- .NET SDK 8+
- SqlPackage (`dotnet tool install -g microsoft.sqlpackage`)
- Docker (for ACR image builds)

## Demo Flow

### 1. Explore Seed Data

```sql
SELECT * FROM dbo.Category;
SELECT * FROM dbo.Product;
SELECT * FROM dbo.Customer;
SELECT * FROM dbo.Review;
```

### 2. Chunk Reviews (AI_GENERATE_CHUNKS)

```sql
EXEC dbo.ChunkReviews @BatchSize = 5;
SELECT * FROM dbo.ReviewVector;
```

### 3. Embed Single Chunk (AI_GENERATE_EMBEDDINGS)

```sql
EXEC dbo.EmbedSingleReview @ReviewVectorId = 1;
```

### 4. Batch Embed All Pending

```sql
EXEC dbo.EmbedReviews @BatchSize = 50;
```

### 5. Semantic Search (VECTOR_SEARCH)

```sql
EXEC dbo.SearchByText @SearchText = N'father and son bonding over hobbies';
```

### 6. DiskANN Vector Index (requires ≥4 vCores)

```sql
EXEC dbo.ToggleVectorIndex @Enable = 1;
EXEC dbo.SearchByText @SearchText = N'craftsmanship and quality';
EXEC dbo.ToggleVectorIndex @Enable = 0;
```

### 7. Trigger: Auto Re-Chunk on Update

```sql
UPDATE dbo.Review SET ReviewText = N'Updated review text.' WHERE ReviewId = 1;
SELECT * FROM dbo.ReviewVector;  -- chunks regenerated by trigger
```

### 8. Re-Embed After Trigger

```sql
EXEC dbo.EmbedReviews @BatchSize = 5;
```

### 9. Change Tracking + Azure Function

```sql
EXEC dbo.ToggleChangeTracking @Enable = 1;
```

### 10. Reset Demo

```sql
EXEC dbo.ResetDemo;
```

## DAB MCP Endpoint

Data API Builder exposes REST, GraphQL, and MCP endpoints for all database objects. After deployment:

- **Health:** `<DAB_URL>/health`
- **REST:** `<DAB_URL>/api/<entity>`
- **GraphQL:** `<DAB_URL>/graphql`
- **MCP:** `<DAB_URL>/mcp`

To connect Copilot to the MCP endpoint, add to `.vscode/mcp.json`:
```json
{
  "servers": {
    "ai-demo-db": {
      "url": "<DAB_URL>/mcp",
      "type": "http"
    }
  }
}
```

## Database Objects

| Object | Type | Purpose |
|--------|------|---------|
| Category, Product, Customer, Review | Tables | Core domain (Star Trek toy shop) |
| ReviewVector | Table | Chunks + VECTOR(1536) embeddings |
| ReviewsPendingEmbedding | View | Chunks without embeddings |
| ChunkReviews | Proc | AI_GENERATE_CHUNKS with fixed-size overlap |
| EmbedSingleReview | Proc | AI_GENERATE_EMBEDDINGS for one chunk |
| EmbedReviews | Proc | Batch embed all pending chunks |
| SearchByText | Proc | Semantic search via text → vector → VECTOR_SEARCH |
| SearchByVector | Proc | Nearest-neighbor search by raw vector |
| ToggleVectorIndex | Proc | Create/drop DiskANN vector index |
| ToggleChangeTracking | Proc | Enable/disable CT on ReviewVector |
| ResetDemo | Proc | Reset database to initial seed state |
| ReviewChanged | Trigger | Auto re-chunks reviews on INSERT/UPDATE |
| text-embedding-3-large | External Model | Azure OpenAI registration |
